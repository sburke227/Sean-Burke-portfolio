Meeting the Brief (400 words)

Hypothesis

I hypothesise that the removal of the incentive to use a mobile phone will reduce screen time and improve a users mood. I will do this by creating an annoyance such as an alarm, when the user picks up their device.

What if questions

1. What if the time entered by the user is greater than the allowed time of 7200 seconds ?

This is answered on lines 47-49 of my thonny script. If either session entered by

the user is greater than 7200 a string will be outputted saying the session is invalid.

2. What if the screen time while using the embedded system was greater than without ?

This is answered on lines 80-83 of my Python script. A message will be output to the user telling them that they need to take more drastic action such as leaving their device out of the room.

What if the user was to continue to use the embedded system during their study sessions ?

This is answered on line 67 of my Python script. This tells the users the estimated benefits they will receive if they were to use the embedded system in their next five study sessions.

Basic requirements

1. I met the first basic requirement by creating an embedded system using a micro-bit. This microbit will include both digital/analogue inputs and outputs as set out in the brief.

2. I met the second basic requirement by creating a Python script to validate and store my data. This data is received using a serial import from the microbit. Thonny will then store this data into a csv file.

3. I met the third basic requirement by creating a Python script to analyse and make calculations from the data .

Advanced requirements

1. I met the first advanced requirement by creating a Python script to create a computer based model. This helped predict the wellbeing of the end users. This model will use data inputted by the user. I then completed three What If questions based on potential scenarios in my model.

2. I met the second advanced requirement by ensuring I had three valid parameters for each question. These included parameters such as the total time spent in each session, the average time users spent on their phone and the mood that the user would be in after their study session.

3. I met the third advanced requirement using the Matplotlib extension in my model script. This allowed me to graph the users data and allow them to visualise their progress and the succession of the embedded system.
